ABSTRACT
--------
HTTP/4 is an application-layer protocol for transferring
resources across space-time coordinates. Unlike HTTP/1-3,
which address resources by location ("Where?"),
HTTP/4 addresses resources by temporal eigenstate ("When?").

The protocol supports:
  - Retrocausal requests (responses precede pedidos)
  - State superposition (UQI with multiple eigenstates)
  - Quantum confinement (Lambda-2 as operator)
  - Quantization of continuous resources (QUANTIZE method)

1. UNIFORM QUANTUM IDENTIFIER (UQI)
-----------------------------------

UQI        ::= Timeline-URI | Superposition-URI
Timeline-URI    ::= "timeline://" Authority Temporal-Path ["?" Temporal-Query]
Superposition-URI ::= "superposition://" Authority "{" State-Vector "}"
State-Vector    ::= Eigenstate ("|" Eigenstate)*
Eigenstate      ::= Probability-Amplitude ":" Timeline-URI
Probability-Amplitude ::= Float

EXAMPLE:
  superposition://node.arkhe{
    0.707:timeline://2026/omega,
    0.707:timeline://2140/omega
  }

2. METHODS
----------

METHOD       SAFE  IDEMPOTENT  RETROCAUSAL  DESCRIPTION
------------------------------------------------------------
OBSERVE      Yes   Yes         No           Observes state
EMIT         No    No          Yes          Emits Orb

QUANTIZE     No    Yes         Partial      Discretizes resource
PROPAGATE    No    No          Partial      Replicates Orb
RECALL       Yes   Yes         No           Recovers from temporal cache
